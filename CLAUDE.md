## What This Is

An RLHF/DPO training data platform: collect human preference feedback (rankings, scalar rewards, binary labels, critiques) on LLM outputs, compute quality scores with inter-annotator agreement, and export RL-ready datasets. Full stack: FastAPI backend, React frontend, Redis worker, PostgreSQL, Prometheus/Grafana observability.

## Commands

### Full stack (Docker)
```bash
docker compose up --build          # All services: API, frontend, worker, postgres, redis, prometheus, grafana
```

### Backend development
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000   # Requires DATABASE_URL and REDIS_URL env vars
python -m workers.quality_worker         # Run quality worker separately
```

### Frontend development
```bash
cd frontend
npm install
npm run dev        # Vite dev server on :5173
npm run build      # TypeScript check + Vite build
```

### Tests
```bash
cd backend
pytest tests/ -v                    # Full suite
pytest tests/test_tasks.py -v       # Single file
pytest tests/test_tasks.py::test_create_task -v   # Single test
```
Tests use SQLite (aiosqlite) + mocked Redis — no external services needed. `asyncio_mode = auto` is set in `pytest.ini`. The test client fixture lives in `backend/tests/conftest.py` and patches all Redis functions + export builder.

## Architecture

### Backend (`backend/`)

**Entrypoint**: `main.py` — FastAPI app with lifespan handler (inits DB tables, pings Redis), CORS for `:3000`/`:5173`, Prometheus instrumentator on `/metrics`.

**Data flow**: API routes handle requests → SQLAlchemy async sessions via `get_db` dependency → models persisted to PostgreSQL. Feedback submission triggers inline quality scoring (Fleiss' kappa, consensus reward, composite quality score) and publishes events to Redis Pub/Sub channel `rl:events:feedback` for the quality worker.

**Key patterns**:
- All DB access is async (SQLAlchemy 2.0 async, `asyncpg` driver). Tests swap to `sqlite+aiosqlite`.
- Redis serves dual purpose: Pub/Sub for real-time quality events + `rl:queue:annotation` list for task assignment queue.
- IDs are UUID strings (`String(36)`), generated by `uuid.uuid4()` in Python (not DB-level).
- The `get_db` dependency auto-commits on success, rolls back on exception.

**Quality scoring** (in `routes/feedback.py`):
- `_fleiss_kappa()` — pairwise agreement from ranking vectors
- `_consensus_reward()` — reliability-weighted average across feedback types
- `_quality_score()` — composite: `0.4*IAA + 0.4*(1-reward_stdev) + 0.2*coverage`
- Tasks auto-complete when `len(feedback) >= min_annotations`

**Workers** (`workers/quality_worker.py`): Redis Pub/Sub consumer that recomputes quality scores. Falls back to polling the Redis queue.

**Exports** (`routes/exports.py`): Generates JSONL (DPO format), Parquet (PyArrow), or HuggingFace Datasets format. Export files written to `EXPORT_DIR` env var.

### Frontend (`frontend/`)

React 18 + TypeScript + Vite. Five pages: Overview (KPIs + Recharts), Tasks (filterable table), Annotate (pairwise ranking UI), Training (metric charts), Exports (dataset builder). API client in `src/api/client.ts` uses React Query for data fetching. Dark-themed design.

### Database models (`backend/models.py`)

Core entities: `Task` → `FeedbackItem` (many), `Task` → `TaskAssignment` (many), `Annotator` → `FeedbackItem` (many), `Dataset` → `TrainingRun`. Enums: `AnnotationType` (ranking/scalar/binary/critique), `TaskStatus` (pending/in_progress/completed/flagged), `TrainingStatus`.

### Test infrastructure

`conftest.py` creates a fresh SQLite DB per test (create_all/drop_all), mocks all Redis operations with in-memory dict/list, and patches both `core.redis_client.*` and route-level imports. The `client` fixture yields an `httpx.AsyncClient` with ASGI transport.
